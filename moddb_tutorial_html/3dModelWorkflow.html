AOE3 3D Model Workflow:

  1. Modeling+UV Mapping

(for the app it's up to you, for this tutorial i'm using autodesk maya 2019)

few things that i think important:

  • use in-game character model/any human model then scale to match 120cm human height as modeling references (very helpful when imported to 3dsmax8, so you dont need to scale a lot to match in-game scale)

  • it's important to use UV space correctly, any UV outside default U1V1 will result in-game black texture. keep in mind, this is an old RTS game, so you can freely adjust UV island proportion based on contribution on rendered pixel. prioritize front-side model face, then the back-side(not noticeable model face) use shared UV space.

  • AOE3 use backface culling, so if there is a mesh, only the normal side will rendered in-game, back-side will be passthrough. so if wanna to create plane mesh, just copy the mesh then invert it, and stack it closely. <todo: add pic in-game engine render backface culling>
  1. Export as OBJ file

  1. Open exported OBJ file to 3dsmax 8 (mandatory, because the gr2 exporter plugin only available on this app)

few things that i think important:

  • Check meshname, sometimes they add "obj" prefix, so fix it. Because when exported .GR2 format save meshname as bone name. this is important for attachment coding.

  • Object pivot, if your imported model show pivot like that, adjust it to center your model. Because when exported, .GR2 format save object pivot as a bone, and using it for in-game placement

  • Flipped normal, sometimes 3dsmax8 show inverted normal, it cant get rid out by re-export from modeling app. idk why. so must be fixed using 3dsmax8 internal tool <todo: add video how to>
